What is Server-Side Web Scripting?

Server - Side Web Scripts basically run on a server. 
It is used to interact with the back-end structure of a web application such as
accessing data or information from a database stored in a server so as to facilitate data transfer 
or produce dynamic content to a web resource. 

Why Use Server Side Scripting?

It enables you to produce dynamic web content 

It enables you to customize response for the user

It enables you to gather and retrieve data from the database

It is used to create large web application such as Bank Apps

DIFFERENT SCRIPTING LANGUAGES

There are a lot of web scripting language but here we are going to tackle only some of the most used languages.

Java 

Java can also be used for web applications. Java web applications enable an individual to create dynamic web pages through SERVLETS or JavaServer Pages a.k.a JSP and also incorporating static web resources like HTML pages and styles. 

These web applications can be deployed by archiving them as WAR files. A WAR file or Web Archive File is a complete archive of web application content.

Java Web Standard Technologies

Servlets
Servlets are Java Clases which extends "HttpServlet" which allows a user to create dynamic and interactive web pages through HTTP request and HTTP response .

Java Server Pages
JavaServer Pages (JSP) are files which holds combined HTML and Java code. JSP is basically Java code directly embeded into HTML pages. Having this kind of setup  The web cotainer compiles the JSP into a servlet at the first time the JSP is accessed.

Life Cycle of a JSP 

  • Instantiation (Object of the Generated Servlet is created)
  • Initialization ( jspInit() method is invoked by the container)
  • Reqeust processing ( _jspService() method is invoked by the container)
  • Destroy ( jspDestroy() method is invoked by the container)

PHP

PHP (recursive acronym for PHP: Hypertext Preprocessor) is a widely-used open source general-purpose scripting language that is especially suited for web development and can be embedded into HTML.


PHP Scripts uses Three main areas

  • Server-side scripting

    • Known as the main objective and the most traditional of all the PHP Scripts. PHP parser (CGI or server module), a Web Server and a Web Browser is required in order to execute and run PHP. Web Server because it is needed together with a PHP installation which is connection. Also, you will have the ability to access the PHP program output with a web browser which you can view the PHP page through the Web Server.

  • Command line scripting

    • PHP Script can also generate and execute without using any server or browser but only required a PHP parser to imply the idea for scripts often executed using CRON/Task Scheduler. It can also be utilized for any text processing tasks.

  • Writing desktop applications

    • PHP is not usually the typical language to use in making and generating a desktop application with a GUI. However, you can still able to use it for the client-side applications if you want to be advanced.
