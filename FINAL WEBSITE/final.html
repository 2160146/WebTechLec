WEB SCRIPTING

What is Server-Side Web Scripting?

Server - Side Web Scripts basically run on a server. 
It is used to interact with the back-end structure of a web application such as
accessing data or information from a database stored in a server so as to facilitate data transfer 
or produce dynami content to a web resource. 

Why Use Server Side Scripting?

It enables you to produce dynamic web content 

It enables you to customize response for the user

It enables you to gather and retrieve data from the database

It is used to create large web application such as Bank Apps

DIFFERENT SCRIPTING LANGUAGES

There a re a lot of web scripting language but here we are going to tackle only some of the most used languages.

Java 

A Java web application is a collection of dynamic resources (such as Servlets, JavaServer Pages, Java classes and jars) and static resources (HTML pages and pictures). A Java web application can be deployed as a WAR (Web ARchive) file. A WAR file is a zip file which contains the complete content of the corresponding web application.

Java Web Standards

Servlets
A servlet is a Java class which extends "HttpServlet" and answers a HTTP request within a web container. 

Java Server Pages
JavaServer Pages (JSP) are files which contain HTML and Java code. The web cotainer compiles the JSP into a servlet at the first time the JSP is accessed.
